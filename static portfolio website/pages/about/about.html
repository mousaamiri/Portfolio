ABOUT

THE ENGINEER BEHIND THE CODE

Engineering with Purpose. Driven by Curiosity.

I believe the best code is the code you can explain to someone who didn't write it.

I studied Computer Science at State University, where I first fell in love with the gap between "it compiles" and "it actually works." Coursework in distributed systems and algorithms gave me the fundamentals, but it was late nights debugging side projects that taught me how to think like an engineer.

My first role was as a Backend Developer at Acme Systems, where I helped rebuild a legacy monolith into a set of maintainable services. That project is where I picked up real DevOps instincts — the difference between code that works on my machine and code that survives production.

Today I focus on Cloud Architecture and building systems that scale without drama. I'm drawn to problems at the intersection of performance and simplicity — the kind of engineering that disappears when it's done right.
